Best Practices

  • Use primary buttons for the main action on a page (e.g., "Save", "Submit")
  • Use secondary buttons for less important actions (e.g., "Cancel", "Back")
  • Use danger buttons only for destructive actions (e.g., "Delete", "Remove")
  • Disable buttons when an action cannot be performed, but provide feedback why
  • Keep button text short and action-oriented (verbs work best)
  • Maintain consistent sizing within the same context or form

Event Bus Integration

Buttons in Harmony publish events rather than calling bounded contexts directly. This maintains loose coupling and allows for better debugging and testing.

// Button publishes event button.addEventListener('click', () => { eventBus.publish({ type: 'PlayAudio', payload: { trackId: '123' } }); }); // Bounded Context subscribes eventBus.subscribe('PlayAudio', (event) => { // Handle playback // Publish result event eventBus.publish({ type: 'PlaybackStarted', payload: { trackId: event.payload.trackId } }); });
