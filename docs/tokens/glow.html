Glow Tokens

Glow tokens define luminous effects for focus rings, active states, and selection highlights. These tokens are critical for accessibility (focus indicators) and user feedback (hover, active, selection states). All glows use GPU-accelerated box-shadow for optimal performance.

Focus Ring Glows

Focus rings are accessibility-critical. They must be clearly visible for keyboard navigation.

--glow-focus-subtle
Subtle
Minimal focus ring for secondary elements. 2px ring + 4px blur.
--glow-focus-default
Default
Standard focus ring for most interactive elements. 2px ring + 8px blur.
--glow-focus-strong
Strong
Prominent focus ring for primary actions. 3px ring + 12px blur.
--glow-focus-error
Error
Focus ring for error states or invalid inputs.
--glow-focus-success
Success
Focus ring for success states or valid inputs.
--glow-focus-warning
Warning
Focus ring for warning states or cautionary actions.

Active State Glows

Active state glows provide feedback when elements are pressed or activated.

--glow-active-subtle
Subtle
Minimal active state with inset shadow for depth.
--glow-active-default
Default
Standard active state for buttons and controls.
--glow-active-strong
Strong
Prominent active state for primary actions.

Selection Highlight Glows

Selection glows indicate selected items in lists, grids, or text selections.

--glow-selection-strong
Strong

Multi
Selection glow for multiple selected items.

Usage Example

/* Apply focus glow to button */ .button:focus { outline: none; box-shadow: var(--glow-focus-default); } /* Apply active glow when pressed */ .button:active { box-shadow: var(--glow-active-default); } /* Apply selection glow to selected list item */ .list-item.selected { box-shadow: var(--glow-selection-default); background: var(--color-primary-50); } /* Apply hover glow on hover */ .card:hover { box-shadow: var(--glow-hover-subtle); }
Accessibility Note: Focus rings are critical for keyboard navigation. Never remove focus indicators without providing an alternative. The glow tokens ensure WCAG 2.1 Level AA compliance for focus visibility (2.4.7).

Performance Considerations

  • GPU Acceleration: All glows use box-shadow, which is GPU-accelerated
  • Avoid Animation: Don't animate box-shadow directly; use opacity transitions
  • Layer Promotion: Elements with glows may create new compositing layers
  • Reduced Motion: Blur effects removed in prefers-reduced-motion mode
