Surface State Tokens

Interactive state variations for surface elements

Token Reference

Hover
--surface-hover
rgba(255, 255, 255, 0.08)
Applied when cursor hovers over interactive surface elements. Provides subtle feedback.
Active
--surface-active
rgba(255, 255, 255, 0.12)
Applied during click/press interaction. Stronger visual feedback than hover.
Selected
--surface-selected
rgba(255, 255, 255, 0.16)
Applied to selected or currently active items. Persistent state indication.
Disabled
--surface-disabled
rgba(255, 255, 255, 0.02)
Applied to non-interactive or disabled elements. Reduces visual prominence.

Usage Guidelines

Best Practices

  • Hover: Use for immediate feedback on interactive elements. Should be subtle enough not to distract.
  • Active: Use during the moment of interaction (mousedown, touch). Should be more prominent than hover.
  • Selected: Use for persistent selection state. Should be clearly visible but not overwhelming.
  • Disabled: Use for non-interactive elements. Combine with reduced opacity and disabled text color.

Code Example

.button {
  background: transparent;
  transition: background 0.2s ease;
}

.button:hover {
  background: var(--surface-hover);
}

.button:active {
  background: var(--surface-active);
}

.button[aria-selected="true"] {
  background: var(--surface-selected);
}

.button:disabled {
  background: var(--surface-disabled);
  color: var(--surface-disabled-text);
  cursor: not-allowed;
}
