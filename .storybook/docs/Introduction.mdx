# Harmony Design System

Welcome to the Harmony Design System documentation. This is a comprehensive design system built for high-performance audio applications with GPU-first rendering and WASM-powered processing.

## What is Harmony?

Harmony is a modern design system that combines:

- **Atomic Design Methodology**: Organized from primitives to templates
- **Web Components**: Native, framework-agnostic components with Shadow DOM
- **GPU-First Rendering**: Hardware-accelerated graphics for audio visualization
- **WASM Performance**: Core logic in Rust compiled to WebAssembly
- **Event-Driven Architecture**: Decoupled components communicating via EventBus

## Performance Targets

Harmony is built with strict performance budgets:

- **Render Budget**: Maximum 16ms per frame (60fps)
- **Memory Budget**: Maximum 50MB WASM heap
- **Load Budget**: Maximum 200ms initial load time
- **Audio Latency**: Maximum 10ms end-to-end

## Architecture Overview

```
┌─────────────────────────────────────────┐
│           UI Layer (Vanilla JS)         │
│  Components → EventBus → Bounded Contexts│
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        Core Logic (Rust → WASM)         │
│  Graph Engine, Audio Processing, State  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│      Hardware Acceleration (GPU)        │
│  WebGPU Rendering, Audio Worklets       │
└─────────────────────────────────────────┘
```
